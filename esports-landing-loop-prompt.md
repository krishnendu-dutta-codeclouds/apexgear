# 🎮 User-Centric Esports Ecommerce Landing Page  

You are a senior UX engineer, esports product designer, and frontend developer.

Your task is to create a **USER-CENTRIC esports ecommerce landing page**.

Follow the loop strictly. Do not skip steps.

---

## 🔁 LOOP GOAL

Create a high-conversion, esports-focused ecommerce landing page  
that appeals to gamers, streamers, and competitive players.

---

## PHASE 1 — USER & REQUIREMENT ANALYSIS

1. Identify the target users:
   - Casual gamers
   - Competitive esports players
   - Streamers / creators

2. Analyze user needs:
   - Fast product discovery
   - Performance-focused gear
   - Trust (reviews, specs, esports branding)
   - Mobile + desktop usability

3. Define requirements:
   - Functional requirements (UI sections, CTAs)
   - Non-functional requirements (performance, accessibility, UX)

**Output:**
- User personas
- User pain points
- Clear landing page goals

---

## PHASE 2 — UX & DESIGN

4. Create a landing page structure:
   - Hero section
   - Featured esports gear
   - Categories (Mouse, Keyboard, Headset, Chair, Monitor)
   - Social proof (reviews, pro-player usage)
   - Limited-time offers
   - Newsletter / CTA

5. Describe visual design:
   - Color palette (dark esports theme)
   - Typography
   - Micro-interactions
   - Accessibility considerations

**Output:**
- Section-by-section UX layout
- Design rationale focused on users

---

## PHASE 3 — SELF CRITIQUE

6. Critique the design:
   - Identify UX friction
   - Identify conversion blockers
   - Identify accessibility issues
   - Identify performance risks

7. List weaknesses clearly.

---

## PHASE 4 — DESIGN IMPROVEMENT

8. Improve the design:
   - Fix all identified weaknesses
   - Simplify user flow
   - Improve CTA clarity
   - Optimize mobile-first experience

Explain what was improved and why.

---

## PHASE 5 — IMPLEMENTATION

9. Generate landing page code:
   - Use semantic HTML
   - Modern CSS (Flexbox / Grid)
   - Lightweight JavaScript only if needed
   - Mobile-first and responsive

**Constraints:**
- No heavy frameworks
- Clean, readable code
- Fast loading
- Gamer-centric UI

---

## PHASE 6 — CODE REVIEW

10. Review the code:
    - Detect bugs
    - Identify security issues
    - Check performance bottlenecks
    - Validate accessibility (ARIA, contrast)

---

## PHASE 7 — REFACTORING

11. Refactor code:
    - Improve readability
    - Optimize performance
    - Remove unnecessary logic
    - Keep UX intact

---

## PHASE 8 — TESTING & VALIDATION

12. Generate basic test cases:
    - Responsive layout tests
    - CTA interaction tests
    - Accessibility checks

13. Verify requirements:
    - User needs met → PASS / FAIL
    - UX clarity → PASS / FAIL
    - Performance → PASS / FAIL

---

## PHASE 9 — LOOP CONTROL

14. If ANY critical issue exists:
    - Return to the relevant phase
    - Repeat the loop

15. Stop ONLY when:
    - No UX blockers remain
    - No critical bugs remain
    - All requirements PASS

---

## FINAL OUTPUT

Present:
- Final UX summary
- Final landing page React, Tailwindcss
- Test checklist (all PASS)
